GPU Based Game Framework

09 October 2024

I’ve been working on making games that are contained entirely in the contents of a fragment shader. I currently have a semi-scalable framework for computing and rendering game state using only the pixels of a single screen buffer. All calculations are performed on the GPU. All game logic is written in GLSL.

This framework currently supports ~10K mutable variables @ ~40 FPS running on an RTX 3090. Truely ground-breaking performance.

Milestones:

  • Proof of concept demo scenes to verify this is possible.

  • Basic framework for variable decleration and mutation.

  • Simple demo with player driven motion & collision.

  • More portable variable encoding system with support for additional variable types.

  • Custom pre-processor to allow for more convienent variable declerations.

  • Custom shader -> processing -> application -> pipeline to get away from some of the limitations of WebGL & ShaderToy.

  • Complete SineWaveSurfers (the game for which this framework is being built).

  • Clever data embeddings.
